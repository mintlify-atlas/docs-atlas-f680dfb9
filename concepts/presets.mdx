## Overview

Web LUTs includes **6 built-in presets** that provide professional color grading effects out of the box. Each preset is a carefully crafted LUT designed to achieve a specific cinematic or photographic look.

```typescript
import { ColorGrader } from 'web-luts';

// Apply a preset to an image
ColorGrader.apply({
  target: document.getElementById('hero'),
  mode: 'cinematic',
  strength: 0.8
});
```

## Available Presets

The library defines preset names as constants for type safety:

```typescript
// From presets/index.ts:5-12
export const PRESETS = {
  NORMAL: 'normal',
  VINTAGE: 'vintage',
  CINEMATIC: 'cinematic',
  SERIOUS_VINTAGE: 'serious-vintage',
  LIGHTS_OUT: 'lights-out',
  SCI_FI: 'sci-fi',
} as const;
```

### Normal

**Mode**: `'normal'`

The identity LUT - applies no color grading. Useful for:
- Resetting to the original image
- Testing the rendering pipeline
- Establishing a baseline for custom effects

```typescript
ColorGrader.apply({ target: img, mode: 'normal' });
```

**Effect**: No color transformation (input = output)

---

### Vintage

**Mode**: `'vintage'`

Creates a warm, faded film look reminiscent of analog photography from the 1970s-80s.

**Color Characteristics**:
- Warm overall tone with increased reds
- Slightly faded greens
- Reduced blue saturation
- Lifted shadows (faded blacks)

```typescript
// Programmatic fallback algorithm (presets/index.ts:113-118)
case PRESETS.VINTAGE:
  finalR = Math.min(1.0, normalR * 1.1 + 0.05);   // +10% red, lift by 5%
  finalG = Math.min(1.0, normalG * 0.95 + 0.03);  // -5% green, lift by 3%
  finalB = Math.min(1.0, normalB * 0.8);          // -20% blue
  break;
```

**Best for**: Portraits, lifestyle photography, nostalgic content

---

### Cinematic

**Mode**: `'cinematic'`

The Hollywood-standard "orange and teal" look that dominates modern cinema. This color scheme creates visual separation and drama.

**Color Characteristics**:
- Teal/cyan tones in shadows and midtones
- Orange/warm tones in highlights and skin tones
- Enhanced contrast between cool and warm areas
- Increased color separation

```typescript
// Programmatic fallback algorithm (presets/index.ts:120-133)
case PRESETS.CINEMATIC:
  if (normalR + normalG + normalB < 1.5) {
    // Shadows → teal
    finalR = normalR * 0.9;
    finalG = normalG * 1.05;
    finalB = normalB * 1.1;
  } else {
    // Highlights → orange
    finalR = Math.min(1.0, normalR * 1.1);
    finalG = Math.min(1.0, normalG * 1.0);
    finalB = Math.min(1.0, normalB * 0.9);
  }
  break;
```

**Best for**: Movie stills, dramatic photography, action content, sci-fi

---

### Serious Vintage

**Mode**: `'serious-vintage'`

An intensified version of the Vintage preset with stronger color shifts and more pronounced fading.

**Color Characteristics**:
- Strong warm cast (heavy red boost)
- Significantly reduced greens
- Heavily desaturated blues
- More aggressive shadow lifting
- Higher overall contrast

```typescript
// Programmatic fallback algorithm (presets/index.ts:135-140)
case PRESETS.SERIOUS_VINTAGE:
  finalR = Math.min(1.0, normalR * 1.15 + 0.07);  // +15% red, lift by 7%
  finalG = Math.min(1.0, normalG * 0.85 + 0.04);  // -15% green, lift by 4%
  finalB = Math.min(1.0, normalB * 0.7);          // -30% blue
  break;
```

**Best for**: Retro content, deliberately aged looks, sepia-style effects

---

### Lights Out

**Mode**: `'lights-out'`

A dark, moody preset that creates a mysterious nighttime or low-light atmosphere.

**Color Characteristics**:
- Overall darkening of the image
- Cool blue tint in shadows
- Reduced reds (cooler skin tones)
- Muted greens
- Enhanced blues for moonlight effect

```typescript
// Programmatic fallback algorithm (presets/index.ts:142-147)
case PRESETS.LIGHTS_OUT:
  finalR = normalR * 0.85;                        // -15% red
  finalG = normalG * 0.9;                         // -10% green
  finalB = Math.min(1.0, normalB * 1.05 + 0.04);  // +5% blue, lift by 4%
  break;
```

**Best for**: Night scenes, thriller/horror content, moody portraits

---

### Sci-Fi

**Mode**: `'sci-fi'`

A futuristic, cool-toned preset inspired by science fiction films and cyberpunk aesthetics.

**Color Characteristics**:
- Cool blue-green overall tone
- Reduced warm tones (desaturated reds)
- Enhanced greens and cyans
- Boosted blues for a technological feel
- Clean, clinical appearance

```typescript
// Programmatic fallback algorithm (presets/index.ts:149-154)
case PRESETS.SCI_FI:
  finalR = normalR * 0.8;                         // -20% red
  finalG = Math.min(1.0, normalG * 1.05 + 0.03);  // +5% green, lift by 3%
  finalB = Math.min(1.0, normalB * 1.1 + 0.05);   // +10% blue, lift by 5%
  break;
```

**Best for**: Tech content, futuristic themes, cyberpunk aesthetics, medical/clinical imagery

## Loading Presets

Presets are loaded asynchronously from `.cube` files:

```typescript
import { loadPreset } from 'web-luts/presets';

// Load a single preset
const vintageData = await loadPreset('vintage');

// Preload all presets for instant switching
import { preloadAllPresets } from 'web-luts/presets';
preloadAllPresets();
```

### Load Flow

The `loadPreset` function (presets/index.ts:34-88) follows this hierarchy:

1. **Check Cache**: Return immediately if already loaded
2. **Fetch CUBE File**: Attempt to load from URL
3. **Parse & Validate**: Ensure data integrity
4. **Fallback**: Use programmatic LUT if file fails
5. **Cache**: Store for future use

```typescript
// From presets/index.ts:41-52
try {
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`Failed to fetch LUT: ${response.statusText}`);
  }
  
  const text = await response.text();
  const lut = parseCubeFile(text);
  
  // Validate
  const expectedEntries = lut.size * lut.size * lut.size * 3;
  if (lut.data.length !== expectedEntries) {
    throw new Error('Incomplete LUT data');
  }
```

## Programmatic Fallbacks

If CUBE files fail to load (network error, missing files, etc.), Web LUTs generates **programmatic fallback LUTs** that approximate the intended effect.

### Why Fallbacks?

- **Reliability**: App continues working even if assets fail
- **Offline Support**: No network dependency
- **Development**: Works before CUBE files are created
- **Graceful Degradation**: Users see _some_ effect rather than errors

### Algorithm

The `createProgrammaticLut` function (presets/index.ts:95-165) generates a synthetic LUT:

```typescript
function createProgrammaticLut(name: PresetName): ParsedLut {
  const size = 17;
  const data = new Float32Array(size * size * size * 3);
  let i = 0;
  
  for (let b = 0; b < size; b++) {
    for (let g = 0; g < size; g++) {
      for (let r = 0; r < size; r++) {
        const normalR = r / (size - 1);
        const normalG = g / (size - 1);
        const normalB = b / (size - 1);
        
        // Apply preset-specific color transformations
        let finalR = normalR;
        let finalG = normalG;
        let finalB = normalB;
        
        switch (name) {
          case PRESETS.VINTAGE:
            // ... transformation logic
          case PRESETS.CINEMATIC:
            // ... transformation logic
          // etc.
        }
        
        data[i++] = finalR;
        data[i++] = finalG;
        data[i++] = finalB;
      }
    }
  }
  
  return { size, data, title: name };
}
```

<Note>
  Programmatic fallbacks provide approximate effects. For production use, always include the actual CUBE files for best quality.
</Note>

## Preset Caching

Presets are cached after first load to improve performance:

```typescript
// From presets/index.ts:27-38
const presetCache = new Map<PresetName, ParsedLut>();

export async function loadPreset(name: PresetName): Promise<ParsedLut> {
  // Return from cache if available
  if (presetCache.has(name)) {
    return presetCache.get(name)!;
  }
  
  // ... load and parse ...
  
  // Cache for future use
  presetCache.set(name, lut);
  return lut;
}
```

**Benefits**:
- Faster preset switching (no re-fetching)
- Reduced network traffic
- Smoother user experience

## Validation

The library validates preset names at runtime:

```typescript
// From presets/index.ts:172-174
export function isValidPreset(name: string): name is PresetName {
  return Object.values(PRESETS).includes(name as PresetName);
}
```

Usage:

```typescript
const userInput = 'cinematic';

if (isValidPreset(userInput)) {
  await loadPreset(userInput);
} else {
  console.error('Invalid preset name');
}
```

## Preloading Strategy

For optimal UX, preload all presets at app startup:

```typescript
// From presets/index.ts:179-185
export function preloadAllPresets(): void {
  Object.values(PRESETS).forEach(preset => {
    loadPreset(preset as PresetName).catch(() => {
      // Errors already logged in loadPreset
    });
  });
}
```

Call this early in your app initialization:

```typescript
import { preloadAllPresets } from 'web-luts/presets';

// In your app's entry point
preloadAllPresets();
```

**Result**: Instant preset switching with no loading delay

## Blending Strength

All presets support adjustable strength (0-1) to control the intensity:

```typescript
// Subtle effect (20%)
ColorGrader.apply({ 
  target: img, 
  mode: 'cinematic', 
  strength: 0.2 
});

// Full effect (100%)
ColorGrader.apply({ 
  target: img, 
  mode: 'cinematic', 
  strength: 1.0 
});
```

The shader blends between the original and graded colors:

```glsl
vec3 finalColor = mix(color.rgb, gradedColor, uStrength);
```

## Custom Presets

While the library includes 6 built-in presets, you can load your own:

```typescript
import { ColorGrader } from 'web-luts';

// Load a custom CUBE file
const customHandle = await ColorGrader.loadCustomCube('/luts/my-custom.cube');

// Apply it like a built-in preset
ColorGrader.apply({ 
  target: img, 
  mode: customHandle 
});
```

See the [Custom LUTs guide](/guides/custom-luts) for more details on creating and using custom color grades.

## Performance Notes

| Operation | Time | Notes |
|-----------|------|-------|
| **First Load** | ~50-100ms | Fetch + parse CUBE file |
| **Cached Load** | Less than 1ms | Retrieved from memory |
| **Preset Switch** | ~5-10ms | WebGL texture binding |
| **Fallback Generation** | ~10-20ms | One-time computational cost |
